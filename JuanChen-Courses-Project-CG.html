High-performance computers are mostly used to solve large-scale scientific numerical calculation problems. The calculation of them is generally very large. For example, in the field of complex biomolecule simulation, the calculation has reached million trillion times. For the practical application of the problem, the calculation is more than 100M times. In the existing application requirements, high-performance computer programs often need to be run for several days or even months continuously. For example, the protein folding program in the blue gene developed by IBM needs to run for several months. When large-scale parallel computer calculates the larger data, it may lose data due to the exit calculation caused by the failure of calculation. For a large-scale, which needs hundreds of hours to calculate, reloading calculation is costly.
In order to ensure that the scientific computing program can be completed correctly on a large-scale system and improve the reliability of the system, large-scale systems must be fault tolerant to hardware failure. Fault tolerance is the ability of computers to tolerate failures using more resources. Even in the event of a failure, the program does not need to re-operation. The way to implement hardware fault tolerance in current large-scale system is to roll back the recovery through software. The principle is as follows: When a parallel job is running, the state of the calculation at a certain time is called a checkpoint and is stored in this state; in the event of a failure, the parallel job is restarted and rolled back to the nearest one checkpoint to recalculate the task from the last checkpoint to the fault.
Checkpoint technology achieves fault detection through recalculating the task from the last checkpoint to the position of fault. Therefore, it is very important to study the software fault tolerant technology, which is very important to solve the reliability problem of large-scale system.
